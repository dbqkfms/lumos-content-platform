# LUMOS UX Audit and Streaming Notes

## 사용자 관점 점검 포인트

### 1. 첫 화면
- 장점
  - 브랜드 인상은 강하다.
  - `LED + 콘텐츠 + 설치 시뮬레이션` 메시지가 이전보다 명확하다.
- 남은 체크
  - 모바일 메뉴는 실제 기기에서 한 번 더 확인 필요
  - 홈 영상의 첫 프레임이 너무 어두우면 로딩 초반 인상이 약해질 수 있음

### 2. 구조
- 현재 구조는 오픈용으로 적절하다.
  - Home
  - Standard
  - Local
  - Artwork Detail
  - Process
- 불필요하게 페이지를 더 늘리기보다, 이 5개를 더 탄탄하게 하는 게 낫다.

### 3. 줄바꿈 / 카피
- Standard 설명은 짧게 유지하는 편이 좋다.
- Local은 “예술성”보다 “오래 틀어도 편안함”을 먼저 말해야 한다.
- Artwork Detail에서는 “이 작품을 사고 싶나요?”보다 “이 작품을 공간에 어떻게 적용할지 제안”이 더 맞다.

### 4. 이미지 / 장면 인상
- 첨부된 Genspark 화면 기준으로 우선순위가 높은 자산
  - 금색 구형 오브제
  - 월광 회랑형 장면
  - 링 포털 장면
  - 백색 연꽃 장면
  - 궁궐 + 달 장면
  - 외벽 코너 LED 장면
  - 실내 파노라마 LED 월 장면
- 이유
  - 멀리서도 읽히는 단일 주제
  - 검은 여백과 강한 대비
  - 실제 LED 적용 상상이 쉬움
  - 사람 스케일이 들어가 있어 크기감 설명이 쉬움

### 5. 아직 더 손봐야 할 것
- 1000+ 내부 자산 규모를 보여주는 섹션
- THISGLOBAL 본사이트와 문구 톤 통일
- 문의 폼 라우팅 문구
- 작품별 태그 정밀도

## 송출 시스템 초안

### 오픈 직후 표현
- 현재는 `실시간 송출 플랫폼`보다 `납품형 + 운영지원형`으로 표현
- 고객에게는 아래처럼 설명
  - 화면 환경 분석
  - 콘텐츠 선정
  - 설치 시뮬레이션
  - 납품용 파일 패키지 제공
  - 운영지원 / 업데이트 대응

### 내부 구성 권장
- 콘텐츠 원본 저장소
  - master video
  - display-fit variants
  - preview
  - thumbnail
- 메타데이터 레이어
  - world
  - tags
  - space fit
  - display fit
  - ops flags
- 납품 패키저
  - landscape
  - portrait
  - ribbon
  - facade
- 운영 이력
  - 어느 공간에 무엇이 납품됐는지
  - 수정/업데이트 요청
  - 재송부 이력

### 1차 실제 운영 방식
- USB / 로컬 플레이어 납품
- 디스플레이 CMS가 있는 고객은 CMS 호환형 파일 납품
- 월드/공간별 플레이리스트 제안서 제공
- 고객 요청 시 업데이트 파일 재전달

### 2차 확장 방향
- 원격 업데이트
- 플레이리스트 스케줄
- 화면별 버전 자동 매핑
- 운영 로그와 CS 연동

### CS 접점 권장
- 문의 유형을 최소 4개로 분리
  - LED 견적
  - 콘텐츠 추천
  - 통합 제안
  - 운영 / 수정 요청
- 접수는 하나로 받고 내부에서 분기
  - THISGLOBAL
  - LUMOS
